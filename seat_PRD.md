# 🏫 교실 관리 프로그램 PRD (Product Requirements Document)
## `seat.html` — 자리 변경 & 발표 뽑기 통합 프로그램

---

## 📋 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **프로젝트명** | 교실 매직박스 (Classroom Magic Box) |
| **대상** | 초등학교 6학년 교실 (약 25~30명) |
| **사용자** | 교사 (관리자), 학생 (관람자) |
| **파일 형태** | 단일 HTML 파일 (`seat.html`) |
| **기술 스택** | HTML + CSS + Vanilla JavaScript (외부 의존성 없음) |
| **핵심 가치** | 긴장감 있는 연출 + 교사 편의성 + 공정성(과 몰래 조작 가능성) |

## 🎯 핵심 기능 요약

### A. 자리 변경 프로그램
- 학생 명단 입력 및 교실 배치도 설정
- 성별 고정 (남/여 좌석 지정)
- 특정 학생 자리 고정 (시력, 키 등 사유)

- **시크릿 모드**: 교사가 몰래 특정 학생을 지정하여 뽑기
- 중복 방지 옵션
- 뽑기 히스토리 관리

# 📐 1단계: 기반 구축 (Foundation)

> **목표**: 학생 데이터 관리 시스템과 기본 UI 프레임워크 구축

### 1-1. 메인 화면 구조

```
┌─────────────────────────────────────┐
│         🎪 교실 매직박스              │
│                                     │
│   ┌─────────┐    ┌─────────┐        │
│   │  🪑     │    │  🎤     │        │
│   │ 자리    │    │ 발표    │        │
│   │ 변경    │    │ 뽑기    │        │
│   └─────────┘    └─────────┘        │
│                                     │
│         ⚙️ 학생 관리                 │
└─────────────────────────────────────┘
```

### 1-2. 학생 데이터 관리

**입력 방식 (3가지 지원)**

1. **직접 입력**: 이름 + 성별(남/여) 개별 추가
2. **일괄 입력**: 텍스트 영역에 줄바꿈으로 구분하여 붙여넣기
   - 형식: `이름, 성별` (예: `김민준, 남`)
3. **localStorage 저장/불러오기**: 한 번 입력하면 브라우저에 저장

**학생 데이터 구조**

```javascript
{
  id: 1,
  name: "김민준",
  gender: "M",          // M: 남, F: 여
  fixedSeat: null,      // 고정 좌석 번호 (null이면 미고정)
  genderLock: false,    // 성별 고정 적용 여부
  isPresent: true,      // 출석 여부
  pickCount: 0          // 발표 뽑힌 횟수
}
```

### 1-3. 교실 배치도 설정

- 행(가로) × 열(세로) 격자 설정 (기본값: 5×6)
- 빈자리(사용 안 함) 지정 가능
- 교탁 위치 표시 (상단 고정)
- 드래그로 좌석 비활성화/활성화 토글

### 1-4. UI/UX 디자인 방향

| 요소 | 방향 |
|------|------|
| **테마** | 놀이공원/마술쇼 컨셉 — 어둡고 화려한 배경 |
| **색상** | 다크 네이비 배경 + 네온 골드/시안 포인트 |
| **폰트** | 제목: 둥근 디스플레이체 / 본문: 깔끔한 고딕 |
| **분위기** | 교실 TV에 띄워놓으면 아이들이 "와!" 하는 느낌 |

### 1-5. 1단계 산출물

# 🪑 2단계: 자리 변경 프로그램 (Seat Shuffle)

> **목표**: 긴장감 넘치는 셔플 연출 + 교사 맞춤 제약조건 시스템

### 2-1. 제약조건 설정 패널

**A. 성별 고정 (Gender Lock)**

```
┌─ 성별 고정 설정 ──────────────────────┐
│                                       │
│  모드 선택:                            │
│  ○ 사용 안 함                          │
│  ○ 남녀 영역 분리 (왼쪽/오른쪽)         │
│  ○ 남녀 짝꿍 배치 (남-여 교차)          │
│  ○ 특정 열만 성별 지정                  │
│                                       │
│  [열 1: 남] [열 2: 무관] [열 3: 여]...  │
└───────────────────────────────────────┘
```

- 남녀 영역 분리: 교실을 좌/우로 나누어 남학생·여학생 배치
- 남녀 교차 배치: 짝꿍이 반드시 다른 성별
- 열 단위 지정: 특정 열에 특정 성별만 배치

**B. 특정 학생 고정 (Seat Pin)**

- 교실 배치도에서 좌석 클릭 → 학생 선택 → 고정
- 고정 사유 메모 기능 (시력 저하, 집중력, ADHD 등)
- 고정된 좌석은 📌 아이콘으로 시각 표시
- 복수 학생 동시 고정 가능

**C. 이웃 제한 (선택적 고급 기능)**

- "A와 B는 인접하지 않게" 규칙 추가 가능
- 최대 5개 규칙까지 설정

### 2-2. 셔플 애니메이션 연출

**단계별 연출 시나리오**

```
[STEP 1] 준비 — 3, 2, 1 카운트다운 (화면 전체 + 효과음 옵션)
    ↓
[STEP 2] 해체 — 모든 이름표가 자리에서 떠오름 (부유 애니메이션)
    ↓
[STEP 3] 셔플 — 이름표들이 화면 중앙에서 빠르게 뒤섞임
         (3~5초간 회전 + 색상 변화 + 속도 변화)
    ↓
[STEP 4] 착지 — 하나씩 새 자리로 날아가며 안착
         (앞자리부터 순서대로, 또는 랜덤 순서)
    ↓
[STEP 5] 공개 — 최종 배치 완성 + 축하 이펙트
```

**애니메이션 세부 사항**

- 셔플 중: 이름표가 빠르게 위치를 교환하는 모습 (읽을 수 없을 정도로 빠르게)
- 착지 시: 한 명씩 "뿅!" 하고 자리에 안착 (bounce 효과)
- 완료 시: 반짝이/confetti 효과
- **속도 조절 슬라이더**: 교사가 연출 시간 조절 가능 (빠르게 3초 ~ 느리게 15초)

### 2-3. 결과 화면

```
┌──────────────────────────────────────┐
│            [ 교 탁 ]                  │
│                                      │
│  [김민준] [이서연] [박지호] [최유나]    │
│  [정하준] [강수빈] [윤도현] [임지우]    │
│  [조현우] [한예린] [송민서] [배나영]    │
│  [📌고정] [오승현] [류하은] [신준혁]    │
│  [서지안] [문태현] [장소율] [권도윤]    │
│                                      │
│  [다시 섞기]  [결과 저장]  [인쇄]      │
└──────────────────────────────────────┘
```

- 색상 구분: 남학생(파란 계열), 여학생(분홍/보라 계열), 고정(금색 테두리)
- 이전 자리 → 새 자리 이동 경로 표시 (토글 가능)
- 결과를 이미지로 저장 (캡처 기능)
- 인쇄용 레이아웃 (심플 흑백 버전)

### 2-4. 2단계 산출물

- [x] 성별 고정 모드 3종 (분리/교차/열 지정)
- [x] 특정 학생 좌석 고정 + 메모
- [x] 5단계 셔플 애니메이션 시퀀스
- [x] 속도 조절 슬라이더
- [x] 결과 저장/인쇄 기능
- [x] 셔플 알고리즘 (Fisher-Yates + 제약조건 충족 검증)

# 🎤 3단계: 발표 뽑기 프로그램 (Random Picker)

> **목표**: 캡슐머신 연출 + 교사 시크릿 모드 구현

### 3-1. 캡슐머신 UI

```
        ┌─────────────┐
        │  🎪 발표자   │
        │    뽑기!     │
        │             │
        │   ┌─────┐   │
        │   │ ○○○ │   │  ← 캡슐들이 통 안에서 움직임
        │   │○ ○○ │   │
        │   │ ○ ○○│   │
        │   └──┬──┘   │
        │      │      │
        │   [출구]     │  ← 여기서 캡슐이 나옴
        │             │
        │  ┌────────┐  │
        │  │ ?????? │  │  ← 결과 표시 영역
        │  └────────┘  │
        └─────────────┘
```

**캡슐 디자인**

- 각 캡슐 = 학생 1명 (색상은 랜덤 배정)
- 캡슐 위에 번호만 표시 (이름은 뽑힌 후 공개)
- 통 안에서 캡슐들이 실시간으로 움직이는 물리 시뮬레이션
- 교사가 **캡슐을 직접 클릭하여 선택** 가능

### 3-2. 뽑기 연출 시나리오

```
[STEP 1] 흔들기 — 교사가 "뽑기" 버튼 클릭 or 캡슐 직접 선택
    ↓
[STEP 2] 섞기 — 캡슐들이 통 안에서 격렬하게 뒤섞임 (2~4초)
    ↓
[STEP 3] 낙하 — 하나의 캡슐이 출구로 떨어짐
    ↓
[STEP 4] 개봉 — 캡슐이 열리며 이름 공개!
         (드럼롤 + 조명 효과 + 이름 확대 표시)
    ↓
[STEP 5] 결과 — 화면에 크게 이름 표시 + 효과
```

### 3-3. 뽑기 모드

| 모드 | 설명 |
|------|------|
| **자동 뽑기** | 버튼 클릭 → 완전 랜덤 선택 |
| **캡슐 선택** | 교사가 통 안의 캡슐을 직접 클릭 |
| **연속 뽑기** | N명을 연속으로 뽑기 (모둠 구성용) |
| **⭐ 시크릿 모드** | 아래 상세 설명 |

### 3-4. ⭐ 시크릿 모드 (Secret Mode) — 핵심 기능

> 교사가 학생들 몰래 특정 학생을 "당첨"시키는 기능

**진입 방법 (학생들에게 들키지 않도록)**

- 화면 우측 하단 빈 공간을 **5번 연속 탭** → 시크릿 패널 오픈
- 또는 키보드 단축키: `Ctrl + Shift + S`
- 패널은 작은 오버레이로 표시, 교사 노트북에서만 보이도록 위치 조정

**시크릿 패널 UI**

```
┌─ 🔒 시크릿 모드 ─────────────┐
│                               │
│  다음 뽑기 결과 지정:          │
│  ┌──────────────────┐        │
│  │ 학생 검색/선택 ▼  │        │
│  └──────────────────┘        │
│                               │
│  ☑ 1회만 적용 후 자동 해제     │
│  ☐ 해제할 때까지 유지          │
│                               │
│  [적용]  [취소]               │
│                               │
│  ※ 화면에 어떤 표시도 남지     │
│    않습니다                    │
└───────────────────────────────┘
```

**작동 방식**

1. 시크릿 모드 활성화 시, 다음 뽑기의 결과가 미리 결정됨
2. 셔플 애니메이션은 **완전히 동일하게** 재생 (학생은 차이를 느낄 수 없음)
3. 캡슐 선택 모드에서도 작동: 어떤 캡슐을 선택하든 지정된 학생이 나옴
4. 적용 후 자동으로 해제 (1회성) 또는 수동 해제 선택 가능
5. 브라우저 콘솔에도 흔적이 남지 않도록 처리

**사용 시나리오 예시**

- 발표 기회를 공평하게 주고 싶을 때
- 자신감이 부족한 학생에게 기회를 주고 싶을 때
- 수업 흐름상 특정 학생이 발표해야 할 때

### 3-5. 추가 기능

**뽑기 히스토리**

- 오늘 뽑힌 학생 목록 표시
- 중복 뽑기 방지 토글 (이미 뽑힌 학생 제외)
- 전체 리셋 버튼

**공정성 보조**

- 각 학생별 뽑힌 횟수 통계 (교사만 확인 가능)
- "오래 안 뽑힌 학생 우선" 모드 (가중치 랜덤)

### 3-6. 3단계 산출물

- [x] 캡슐머신 UI + 물리 시뮬레이션
- [x] 캡슐 직접 선택 기능
- [x] 4종 뽑기 모드 (자동/선택/연속/시크릿)
- [x] 시크릿 모드 (5탭 진입 + 키보드 단축키)
- [x] 뽑기 애니메이션 5단계 시퀀스
- [x] 뽑기 히스토리 + 중복 방지
- [x] 공정성 통계 대시보드

## 🔧 기술 명세 요약

### 파일 구조

```
seat.html (단일 파일, 모든 코드 포함)
├── <style>     — 전체 CSS (애니메이션 포함)
├── <div>       — HTML 구조
└── <script>    — JavaScript 로직
    ├── DataManager      — 학생 데이터 CRUD + localStorage
    ├── SeatShuffler     — 자리 배정 알고리즘 + 제약조건
    ├── CapsuleMachine   — 뽑기 로직 + 물리 시뮬레이션
    ├── AnimationEngine  — 셔플/뽑기 애니메이션 관리
    ├── SecretMode       — 시크릿 모드 관리
    └── UIController     — 화면 전환 + 이벤트 처리
```

### 알고리즘

| 기능 | 알고리즘 |
|------|----------|
| 자리 셔플 | Fisher-Yates Shuffle + 제약조건 백트래킹 |
| 캡슐 물리 | 간단한 2D 충돌 시뮬레이션 (requestAnimationFrame) |
| 가중치 랜덤 | 뽑힌 횟수 반비례 가중치 |
| 시크릿 모드 | 결과 사전 결정 + 동일 애니메이션 파이프라인 |

### 브라우저 호환

- Chrome 80+ (교실 TV/노트북 기준)
- 해상도: 1920×1080 기준 최적화, 1280×720까지 반응형 대응

## ⚠️ 주의사항
